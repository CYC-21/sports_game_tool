# Apps Script 部署步驟

## 1. 開啟指令碼編輯器

1. 在瀏覽器開啟你的 Google 試算表（與本專案同一檔案）。
2. 選單：**擴充功能** → **Apps Script**。
3. 若出現舊的 `Code.gs`，可清空後貼上本資料夾的 `Code.gs` 內容；或 **檔案** → **新增** → **指令碼檔案**，命名為 `Code` 亦可（需刪除預設的 `myFunction` 範例避免重複 `doGet`）。

## 2. 試算表 ID（可選）

- **建議**：使用「容器綁定」試算表時，可將 `Code.gs` 開頭的 `SPREADSHEET_ID` 改成空字串 `''`，並改用 `SpreadsheetApp.getActiveSpreadsheet()`（程式已支援：`SPREADSHEET_ID` 為空則讀取目前綁定的試算表）。
- 若腳本是**獨立專案**、未綁定檔案，請保留 `SPREADSHEET_ID = '你的 ID'`。

## 3. 授權

1. 在編輯器按 **執行**，選任一會觸發存取試算表的函式（之後部署後由 `doGet` 觸發亦可）。
2. 依提示完成 **檢閱權限** → 選擇 Google 帳號 → **進階** → **前往…（不安全）**（若顯示）→ **允許**。

## 4. 部署為 Web 應用程式

1. 右上角 **部署** → **新增部署作業**。
2. 類型選 **網頁應用程式**。
3. 建議設定：
   - **執行身分**：我（你的帳號）
   - **具有存取權的使用者**：**所有人**（公開讀取 API；若只要自己測試可改「僅限自己」）
4. **部署**，複製 **網頁應用程式 URL**（形如 `https://script.google.com/macros/s/.../exec`；請勿公開你的實際網址）。

## 5. 測試網址

在瀏覽器開啟（將 `YOUR_EXEC_URL` 換成你的部署 URL）：

| 用途 | 網址 |
|------|------|
| 說明 | `YOUR_EXEC_URL?resource=help` |
| 賽事（含隊名／場地名 JOIN） | `YOUR_EXEC_URL?resource=matches` |
| 場館 | `YOUR_EXEC_URL?resource=venues` |
| 景點 | `YOUR_EXEC_URL?resource=places` |
| 球隊 | `YOUR_EXEC_URL?resource=teams` |
| 球員 | `YOUR_EXEC_URL?resource=players` |
| 球員—球隊關係 | `YOUR_EXEC_URL?resource=relations` |
| 投稿暫存 | `YOUR_EXEC_URL?resource=matches_pending` |

快取時間為 **10 分鐘**（`Code.gs` 內 `CACHE_SECONDS` 可改為 `300` 即 5 分鐘）。

## 6. 前端與 CORS

部分瀏覽器對 `script.google.com` 的 **fetch** 會遇到 CORS。可二選一：

1. **JSONP**：`YOUR_EXEC_URL?resource=matches&callback=myCb`，回傳 `myCb({...});`，以 `<script>` 載入或動態插入 script。
2. **代理**：用自有網域／Cloudflare Worker 轉發並加上 `Access-Control-Allow-Origin`。

## 7. 分頁名稱必須一致

指令碼使用的分頁名稱為（需與試算表下方分頁 **完全一致**）：

`matches`、`venues`、`places`、`teams`、`players`、`player_team_relations`、`matches_pending`

若你改名，請同步修改 `Code.gs` 的 `readSheetObjects_('...')` 與 `SHEETS` 的鍵（或改為常數對照表）。

## 8. POST 投稿（`doPost` → `matches_pending`）

- 更新 `Code.gs` 後，須 **部署** → **管理部署作業** → 選現有 Web App → **編輯** → 版本選 **新版本** → **部署**，投稿才會用到新程式。
- **網址**：與 GET 相同（`…/exec`），改為 **HTTP POST**（請勿公開你的實際網址）。
- **內容**：`application/x-www-form-urlencoded` 或 `application/json`；欄位名稱須與 `matches_pending` **第 1 列** 英文鍵一致。
- **必填**（伺服器會驗證）：`sport`、`home_team_name`、`away_team_name`、`venue_name`、`date`、`time`。
- **可選**：`league_name`、`submitter`、`note`。`id`、`created_at`、`source` 可留空，由腳本自動填。
- **蜜罐**：若 POST 帶非空 `website`，會拒絕（防機器人；請勿在表單顯示欄填寫）。
- 前端 `web/` 已用 **表單 POST 到隱藏 iframe**，無需處理 POST 的 CORS。

## 9. 權限提醒

「具有存取權的使用者」若為 **所有人**，代表任何人皆可 GET 與 POST；若擔心濫用 POST，可改為僅限自己並另闢管道，或之後再加驗證碼／金鑰。
